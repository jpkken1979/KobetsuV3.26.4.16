## Decisiones tecnicas

1. **No tocar auth**: el usuario pidio explicitamente NO modificar nada de auth porque es uso local-only. Por eso BACK-HIGH-1 (`POST /api/backup` sin gating), FRONT-HIGH-1 (`ADMIN_TOKEN` en localStorage) y FRONT-HIGH-2 (Origin/Referer) NO se aplicaron — son issues reales pero el modelo de threat es local-only.

2. **`sanitizeFilename` con doble pasada**: primero reemplaza `[/\\:*?"<>|]` por `_`, despues colapsa `\.{2,}`. El orden importa: `"a/../../b"` → `"a_.._.._b"` → `"a_____b"` (no `"a___b"` como erroneamente esperaba al principio). Documentado en test.

3. **`detect-unsafe-regex` apagada**: la regla es excesivamente conservadora con regex de parsing de PDFs internos (anchors `^`/`$` estrictos limitan backtracking, inputs son outputs propios de PDFKit, no inputs externos). Falsos positivos puros.

4. **`no-explicit-any: error` relajado en tests**: tests usan `as any` para mockear estructuras complejas (Hono Context, Drizzle row mocks). Es legitimo y limitado a archivos `__tests__/`.

5. **Splits con barrel re-exports**: ambos refactors mantienen el archivo original como barrel (`write.ts`, `api-types.ts`) re-exportando desde sub-archivos. Esto preserva 100% backward compat — ningun importador externo necesita cambiar imports.

6. **Test cleanup explicito**: react-testing-library no auto-limpia DOM con vitest sin config global. Agregado `afterEach(() => cleanup())` en cada describe block (Button, StatusBadge).

## Descubrimientos

1. **`detect-unsafe-regex` flaggea regex con anchors**: incluso `^pattern$` con `.+?` non-greedy es flaggeado. La regla mira solo backtracking teorico, no contexto. Si se reactiva, tendria que ser con eslint-disable line-by-line o reemplazar regex con parsers dedicados.

2. **`getByLabelText` con texto duplicado falla**: si un componente tiene `aria-label` Y children con el mismo texto, RTL devuelve "multiple elements found". Solucion: usar `container.querySelector('[aria-label="X"]')` directo cuando hay duplicacion semantica.

3. **TanStack Router routeTree.gen.ts**: requiere ser regenerado para typecheck en CI fresh. No esta en git (`.gitignore`). Puede regenerarse con:
   ```js
   import { Generator, getConfig } from "@tanstack/router-generator";
   const config = await getConfig({...}, process.cwd());
   await new Generator({ config, root: process.cwd() }).run();
   ```
   El plugin de Vite lo hace automatico en dev, pero en CI sin dev server hay que llamarlo manualmente.

4. **`fs.realpathSync` para defensa de symlinks**: la combinacion correcta es `lstatSync` (rechaza symlink) + `realpathSync` (resuelve target real) + comparacion con `realpathSync(DATA_DIR)`. `path.resolve` solo no detecta symlinks porque es puramente lexico.

5. **`document-index.ts:9`, `document-generation.ts:95`, `calendars.ts:26`** YA tenian try/catch — el audit originalmente lo marco como faltante (LOW-1) pero la verificacion mostro que ya estaba protegido. Solo `seed.ts` necesitaba el wrap real.
