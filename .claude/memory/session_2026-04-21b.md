## Decisiones tecnicas

- **Medicion con pdfjs-dist vs scans**: el approach inicial intento comparar PDF "original papel" vs "generado" — ambos eran scans raster sin text layer. Se pivoteo a medir el PDF fresh generado (con text layer nativo PDFKit) para extraer font sizes reales. Lection: siempre medir la FUENTE del output, no representaciones intermedias.

- **Balance vs tamaño maximo**: el usuario explicito que queria "balance total, no empeorar otras cosas". Se subio font sizes en +1pt, no +2pt, para mantener margen para edge cases (nombres largos, 9 shifts, calendars extensos). El auto-shrink de `cell()` (con piso 3pt + clipping safety net) garantiza que los datos nunca se pierden visualmente.

- **Labels uniformes 7.5pt > 7pt diferenciado**: aunque tecnicamente los row labels vs legal labels tienen semantica distinta, visualmente el usuario percibia el mix como desprolijo. Preferimos uniformidad visual sobre distincion semantica sutil.

- **派遣内容 split forzado con \\n**: con wrap automatico PDFKit dividia al azar (3+1 en lugar de 2+2). Insertar \\n explicito es mas determinista.

- **Checkboxes hardcoded**: decision de producto del usuario — en la practica siempre se usan los mismos valores. Se mantienen los campos `data.isKyoteiTaisho` y `data.responsibilityLevel` en la interface KobetsuData para no romper llamadores existentes, pero se ignoran en render. Si en el futuro se necesita variable, revertir es un edit de 2 lineas.

## Descubrimientos

- **scientist agent fue clave**: midio con pdfjs-dist los font sizes reales del PDF generado, detecto que solo 1/20 bloques (就業日 calendar largo) activaba auto-shrink. Sin esto habriamos estado guessing.

- **Auto-shrink solo mide altura, no ancho**: la funcion `cell()` reduce el font size cuando la ALTURA del texto excede la altura de la celda. El ancho lo resuelve `wrap: true` (default) wrappeando a multilinea, lo que a su vez aumenta altura y dispara shrink. El clipping `doc.rect().clip()` es el ultimo safety net.

- **Excel COL_CW proporciones**: el grid esta basado en character widths del Excel original (A=3.625, B-Z=11.375, AA=4.625, total 292.625). Cambiar COL_CW rompe el paralelismo con Excel pero mejora visual. Se documento en comentarios del codigo.

- **Multi-shift cascade**: el `renderMultiShift` tiene una cascada de attempts (diferentes formatos de columna + font size). Ajustando el techo (de 6.5 a 7.5pt) y piso (de 3.5 a 4pt) dieron mas respiro sin romper 9-shift factories.

- **PDF snapshot tests**: los 9 golden hashes en `server/__tests__/__snapshots__/pdf-hashes.json` cambian con cada ajuste visual. Usar `npm run test:pdf-snapshots:update` para regenerarlos.
