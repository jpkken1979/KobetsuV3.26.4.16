## Que se hizo

- **Nuevo componente `src/routes/contracts/-edit-contract-dialog.tsx`**: Dialog con tabs 社員/派遣先 para editar datos del empleado y fábrica desde el detalle de contrato
- **Integracion en `$contractId.tsx`**: Botón "編集" (Pencil icon) junto a PDF生成 en el header; estado `showEditDialog`; render del dialog
- **Fix `cascading-select.tsx`**: Reemplazado regex-parsing de `workHours` por uso directo de `workHoursDay.split("～")` para extraer horarios de inicio y fin
- **Toggle auto-fill en `shouheisha.tsx`**: Switch que controla si los horarios (workStartTime, workEndTime, breakMinutes) se auto-rellenan al seleccionar fábrica

## Decisiones tecnicas

- El dialog usa `api.updateEmployee()` y `api.updateFactory()` directamente (sin hook wrapper) — patron consistente con el resto del codebase
- El toggle de shouheisha controla **solo horarios** — 単価, 業務内容, シフトパターン, Supervisor siempre se auto-rellenan
- Default `autoFillEnabled = true` para backward compat
