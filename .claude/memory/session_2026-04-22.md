## Que se hizo

- **Agregado `toLocalDateTimeStr`** en `contract-dates.ts`: formato `YYYY-MM-DD_HH-mm` (incluye hora/minuto para distinguir generaciones)
- **Fix filename prefix** en `documents-generate-single.ts`: ahora incluye `contract.contractNumber` para evitar colisiones cuando múltiples contratos del mismo día/company/line generan PDFs con nombre idéntico
- **Endpoint `/preview-by-ids`** agregado en `contracts-batch.ts` (re-exportado también en `contracts-batch.ts` para uso de documents-generate.ts)
- **Regenerados PDFs de 8 contratos 川原鉄工所** (KOB-202604-0010 a 0012 + 5 contratos renew con IDs 53-57), todos con billing_rate=1750, timestamp `_2026-04-22_18-23`

- **Lint warning fix**: removido import no usado `toLocalDateStr` de documents-generate-single.ts

- `toLocalDateStr` (existente) = `YYYY-MM-DD`; `toLocalDateTimeStr` (nueva) = `YYYY-MM-DD_HH-mm`
- El prefix del filename ahora es: `companyName_lineName_contractNumber_timestamp`
- 通知書 no es archivo separado — va como página 2 del PDF 個別契約書
