## Que se hizo

- Fix admin routes: rutas `/api/admin/*` devolvían 503 porque tsx no cargaba `.env`

- Fix middleware: bypass de token check para requests desde localhost

## Decisiones tecnicas

- **Bypass localhost en adminGuardMiddleware**: para app local-only, si la request viene
  de localhost (clientId === "local" | "127.0.0.1" | "::1" | "::ffff:127.0.0.1"),
  se permite acceso admin sin token. Acceso remoto sigue requiriendo ADMIN_TOKEN.
  Razón: el usuario es el único que usa la app en local; el token guard es innecesario.

- **`--env-file .env` en tsx**: tsx no auto-carga `.env` — Node 20.6+ lo soporta
  nativamente via flag. Alternativa descartada: dotenv import (más invasivo).

## Descubrimientos

- `tsx watch server/index.ts` NO carga `.env` automáticamente. El ADMIN_TOKEN en `.env`
  nunca llegaba a `process.env` en el servidor, causando que `!ADMIN_TOKEN` sea true
  y todas las rutas admin devuelvan 503.

- El frontend lee el token admin de `localStorage.getItem("adminApiToken")` (src/lib/api.ts:51).
  El usuario nunca había seteado este valor → header `x-admin-token` nunca se enviaba.
  Con el bypass localhost, esto ya no es necesario para uso local.

- La pantalla admin mostraba "TABLES (0)" porque `GET /api/admin/tables` fallaba con 503
  antes de que la query siquiera llegara a la DB.
