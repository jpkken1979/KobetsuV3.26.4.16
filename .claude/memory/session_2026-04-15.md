## Decisiones tecnicas

- **auditLog action para reset**: El enum no tenia "RESET_ALL". Se uso `action: "delete"` con `entityType: "ALL_TABLES"` para diferenciarlo de deletes normales.
- **adminGuard en tests**: La mini Hono app de tests bypassa `adminGuardMiddleware` deliberadamente para testear solo la logica de negocio. El guard en produccion aplica via `api.use("*", ...)` en index.ts.
- **Counts dentro de la transaccion**: Reviewer detecto race condition (counts fuera de transaction). Fix: mover SELECT count() dentro de `sqlite.transaction()` junto con los DELETE.
- **setTimeout cleanup**: useEffect con retorno de `clearTimeout` para evitar memory leak si el componente se desmonta antes de los 1.5s del redirect.
- **ADMIN_TOKEN a nivel de modulo**: `security.ts` lee ADMIN_TOKEN al cargarse, no por request. Implica que cambiar la env var requiere reiniciar el server.

## Descubrimientos

- **drift guard en tests**: El proyecto tiene un test `claude-md-drift.test.ts` que verifica que CLAUDE.md refleja el numero correcto de route files. Agregar `admin-reset.ts` rompio ese test. Fix: actualizar el contador en CLAUDE.md de 29 a 30.
- **Hook de Edit falso positivo**: El PostToolUse:Edit hook reportaba "Edit operation failed" pero los edits eran exitosos. Los tests confirmaron los cambios.
- **breakMinutes en shiftTemplates**: El campo no es integer sino text (`breakTime`). El schema real usa `breakTime: "60分"` en lugar de `breakMinutes: 60`.
