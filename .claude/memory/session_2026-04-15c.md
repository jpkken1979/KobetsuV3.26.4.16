### 2. factory_yearly_config (config anual por línea)
**Propósito:** guardar 就業日テキスト, 休日, 休暇処理, 指揮命令者, 派遣先責任者 por año fiscal (10/1→9/30), para que PDFs de contratos viejos usen los datos correctos del año.

**Backend:**
- `server/db/schema.ts`: tabla `factory_yearly_config` con uniqueIndex(factoryId, fiscalYear)
- `server/services/factory-yearly-config.ts`: CRUD + `getConfigForYear()` + `copyYearlyConfig()` + `getFactoryYearlyConfigSummary()`
- `server/routes/factory-yearly-config.ts`: 6 endpoints (GET/POST/PUT/DELETE + /summary + /copy-to)
- `server/index.ts`: registrado `/factory-yearly-config`
- DB creada manualmente vía Node.js script (db:push tenía conflicto de índices)

**Frontend:**
- `src/lib/api-types.ts`: `FactoryYearlyConfig`, `FactoryYearlyConfigCreate`, `FactoryYearlyConfigUpdate`
- `src/lib/api.ts`: 4 métodos + `getFactoryYearlyConfigSummary` + `copyFactoryYearlyConfig`
- `src/lib/query-keys.ts`: `factoryYearlyConfig.byFactory()`, `.summary()`
- `src/lib/hooks/use-factory-yearly-config.ts`: 4 hooks + `useCopyFactoryYearlyConfig`
- `src/routes/companies/-factory-yearly-config.tsx`: Dialog con CRUD + copy UI (checkboxes de sibling factories + ejecutar copia)
- `src/routes/companies/-factory-card.tsx`: botón 年度 + `FactoryYearlyConfigDialog` (hover actions)

### 3. company_yearly_config (config anual por empresa — cascade)
**Propósito:** campos que son iguales para todas las líneas de la empresa (休日テキスト, 休暇処理, 派遣先責任者) se guardan una sola vez a nivel empresa. Cascade: factory primero → company fallback.

**Backend:**
- `server/db/schema.ts`: tabla `company_yearly_config` con uniqueIndex(companyId, fiscalYear)
- `server/services/factory-yearly-config.ts`: `getCompanyConfigForYear()` + `getCompanyConfigCRUD()`
- `server/routes/company-yearly-config.ts`: 4 endpoints
- `server/index.ts`: registrado `/company-yearly-config`
- `server/services/document-generation.ts`: `buildCommonDataForPDF()` carga ambos configs; cascade factory→company→factories fields
- DB creada manualmente vía Node.js script

**Frontend:**
- `src/lib/api-types.ts`: `CompanyYearlyConfig`, `CompanyYearlyConfigCreate`, `CompanyYearlyConfigUpdate`
- `src/lib/api.ts`: 4 métodos
- `src/lib/query-keys.ts`: `companyYearlyConfig.byCompany()`
- `src/lib/hooks/use-company-yearly-config.ts`: 4 hooks

## Cascade de datos en PDFs

```
factory_yearly_config  ← prioridad (específico por línea/año)
        ↓ si null
company_yearly_config  ← default de empresa/año
        ↓ si null
factories fields       ← fallback legacy
```

## Fiscal year logic
- `getFiscalYear(date)`: `month >= 10 ? year : year - 1`
- Ejemplo: contrato startDate 2024-01-15 → fiscalYear 2023 (enero es año fiscal anterior)
