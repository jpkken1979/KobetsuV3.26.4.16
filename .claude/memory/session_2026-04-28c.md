## Decisiones técnicas

- **No agregar dependencias nuevas**: aprovecho `motion/react` ya presente; no instalar `react-spring`, `framer-motion-numbers` u otros packages.
- **Spotlight con CSS variables vs canvas/svg**: usar `radial-gradient` con coords del mouse via state es 100x más performante que canvas; el `transition-opacity` del wrapper hace fade-in/out cuando entra/sale del hero.
- **Particle burst con `overflow-visible`**: el wrapper del success usa `relative`, pero importante NO setear `overflow-hidden` o `overflow-x-auto` porque trunca las partículas. En el id-generator se agregó explícito `overflow-visible` por eso.
- **Why `useReducedMotion` en TODOS los componentes nuevos**: regla auto-inyectada del proyecto (`CLAUDE.md` UI Accessibility section). Los components retornan `null` o skip animation cuando active.
- **Stats reales vs mock**: usar `getDashboardStats(30)` ya existente — endpoint trae `companies`, `factories`, `activeContracts`, `activeEmployees`. Para ID generator un stat es estático (`"ID形式" = 2`) porque comunica info estructural, no count.
- **Sidebar badge "一括" cuando NO active vs primary cuando active**: el badge transforma su color en activo, lo que evita necesitar ese active dot extra a la derecha — comportamiento consistente con la jerarquía visual.
- **Breadcrumb va al hero, no como nav separado**: simplifica el layout. La página tiene UN hero shell que contiene todo el contexto.
- **Eliminar header interno del factory-generator**: tras refactor, el `BatchPageShell` ya muestra título "工場一括生成" en gradient grande. Mantener el header interno con su propio title sería duplicación visual confusa.
- **Mantener step indicator del id-generator**: el step indicator (input → preview → result) es información FUNCIONAL de progreso, no decorativa. Reemplazo el título textual por un label "進捗 · Progress" en mono — más sutil pero conserva contexto.

## Descubrimientos

- **`motion/react` Variants typing**: definir variants con `as const` + custom prop typed `(i: number)` requiere que el variant function devuelva un object con shape compatible — TypeScript es estricto pero motion lo acepta cuando el helper se llama con `custom={i}`. Patrón en bento-stats-grid.tsx funciona.
- **`useReducedMotion` corte temprano vs prop conditional**: para particles `return null` directo es más limpio que pasar `transition: { duration: 0 }`. El particle burst no aporta info, es solo deleite — saltarlo entero es correcto.
- **`color-mix(in srgb, ...)` en Tailwind v4**: funciona inline en clase `bg-[color-mix(in_srgb,var(--color-status-ok)_18%,transparent)]` pero las clases largas son ilegibles. Mejor extraer a `ACCENT_TOKENS` lookup como en bento-stats-grid.tsx.
- **`hidden` escape secuence en class names de Tailwind**: para tener `_` literal en el value de un arbitrary class, usar `_` raw — ejemplo: `from-[color-mix(in_srgb,...)]`.
