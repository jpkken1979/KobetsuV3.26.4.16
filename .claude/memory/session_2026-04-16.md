## Decisiones tecnicas
- Se uso el stack real del repo (`motion/react`, Tailwind v4, React 19) en vez de seguir literalmente ejemplos pensados para Framer Motion o Next.js.
- La Opcion 3 del dashboard se implemento con un helper reutilizable (`SpotlightPanel`) en vez de copiar efectos aislados en cada card.
- Se mantuvo el trabajo dentro de `src/routes/` para no introducir dependencias nuevas ni tocar el sistema de diseño global.
- Se mantuvo commit local + bundle como salida segura mientras `target` no respondia, y luego se actualizo el remote al repo accesible para completar la publicacion.
- `.mcp.json` se hizo portable con rutas relativas al workspace actual, no con rutas hardcodeadas por maquina.
- El fallo final en `batch-contracts-service.test.ts` se resolvio corrigiendo el test, no el servicio, porque `entityType: "contract"` ya se escribia bien y el problema era una consulta ambigua sobre `audit_log`.
